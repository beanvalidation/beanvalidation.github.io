Bean Validation

Help and contribute

Come and join us!

Bean Validation 2.0 was finalized in 2017 but it doesn't end here, come and join planning the new features of the next revision!

We welcome your contributions of any sort:

  • feedback
  • proposals
  • contributions
  • joining the EG

Feedback

Feedback is more than welcome, either by starting a discussion on our forum or by opening/commenting an issue in our bug tracker.

Proposals

If you think some new feature should be added to Bean Validation or you want to influence the shape of new Bean Validation features, come and join our development mailing list. In addition, you can have a look at the current proposals discussions here.

Contributions

All specification work is done in the open. This does not only include discussions, but also all code and artifacts. Here is a list of all relevant GitHub source repositories:

  • Specification repository
  • Reference implementation repository
  • API repository
  • TCK repository
  • This website source

Interested? Read all about how to:

  • concretely contributing to the specification
  • work with the API and TCK sources
  • improve this website.

Joining the EG

Bean Validation operates under the 2.10 rules of the JCP (Java Community Process).

If you either are or represent a Full member of the JCP, you may become a member of the next JSR expert group. If you are an Associate Member of the JCP, you may become an officially recognized contributor of this JSR. Nominate yourself to get started.
